CKEditor Plug-in → Character Entity Settings

Character Entity Settings

Output HTML entities
Whether to use HTML entities in the editor output. This option reflects the setting CKEDITOR.config.entities in the CKEditor API.
Output math and greek symbols as entities
Whether to convert some symbols, mathematical symbols, and Greek letters to HTML entities. This may be more relevant for users typing text written in Greek. The list of entities can be found in the W3C HTML 4.01 Specification, section 24.3.1. This option reflects the setting CKEDITOR.config.entities_greek in the CKEditor API.
Output Latin1 (ISO 8859-1) entities
Whether to convert some Latin characters (Latin alphabet No. 1, ISO 8859-1) to HTML entities. The list of entities can be found in the W3C HTML 4.01 Specification, section 24.2.1. This option reflects the setting CKEDITOR.config.entities_latin in the CKEditor API.
Output user-defined entities
If this option is checked, then the entities that are defined in the Docmenta configuration (Administration â†’ Entities) are used in the editor output. The listbox allows to define whether the user-defined entities shall be inserted as symbolic or numeric entities. This option reflects the setting CKEDITOR.config.entities_additional in the CKEditor API.
Use numeric entity representation
Whether to output entities as decimal numeric entity. This listbox allows to define whether only remaining non-ASCII characters shall be converted to numeric entities or all entities. This option reflects the setting CKEDITOR.config.entities_processNumerical in the CKEditor API.
