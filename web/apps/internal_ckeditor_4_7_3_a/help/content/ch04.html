HTML5 Figure Tag Support

This option defines, whether CKEditor supports the HTML5 figure and figcaption tags, to create labeled images. If the option is set to "Disabled", then the CKEditor image dialog just allows to insert and edit img tags. In this case, a labeled image has to be created as an img tag with title attribute.
If the option is set to "Enabled", then, in addition to images that are inserted using the img tag only, the CKEditor image dialog allows to create labeled images using the HTML5 figure and figcaption tags.
The settings "Disabled" and "Enabled" affect all products. To enable or disable the figure tag only for selected products, choose either the setting "Disabled for selected products" or "Enabled for selected products".
Products
If the option "Disabled for selected products" is selected, then the figure tag is only disabled for the products listed in the input field below (list of product IDs, separated by space). For all other products the figure tag is enabled. Â 
If the option "Enabled for selected products" is selected, then the figure tag is only enabled for the products listed in the input field below (list of product IDs, separated by space). For all other products the figure tag is disabled.
Figure CSS class
If the figure tag is enabled, then this option allows to define the name of a CSS class to be assigned to every figure element. For example, enter the ID of a Docmenta block-style, to apply the formatting of this style to every created figure element. If not set, then the default CSS class with name "figure" is used. This option reflects the setting CKEDITOR.config.image2_captionedClass in the CKEditor API.
